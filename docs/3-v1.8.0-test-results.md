## Detailed Test Results

### 1. ADF Description Writes — FIXED

New `--body-adf` flag on both `edit` and `comment add` accepts raw ADF JSON:

```bash
echo "" | jira issue edit PO-2701 --body-adf '{"type":"doc","version":1,...}' --no-input
# ✓ Issue updated

echo "" | jira issue comment add PO-2701 --body-adf '{"type":"doc","version":1,...}' --no-input
# ✓ Comment added to issue "PO-2701"
```

Verified: description on Jira preserved the ADF structure (headings, paragraphs, etc.)
with full fidelity. The old `-b` / `--body` flag still exists for plain text but will
fail on API v3 if it sends raw text instead of ADF.

**Caveat**: The plain text `-b` flag is now broken — API v3 rejects non-ADF body.
Agents MUST use `--body-adf` for all writes.

### 2. API Version — FIXED

Debug output confirms v3 for reads:
```
GET /rest/api/3/issue/PO-2701 HTTP/1.1
```

Edit error message confirms v3 for writes:
```
description: Operation value must be an Atlassian Document (see the Atlassian Document Format)
```

The v2/v3 mismatch is resolved.

### 3. Interactive Prompts — PARTIAL

Global default is now non-interactive (good):
```
--interactive  Enable interactive prompts (default: non-interactive for automation safety)
```

**However, all write commands still hang when stdin is a TTY.** The CLI reads stdin
even when all arguments are provided via flags. Workaround: pipe empty stdin.

```bash
# HANGS (no stdin pipe):
jira issue edit PO-2701 -s"New title" --no-input

# WORKS (with stdin pipe):
echo "" | jira issue edit PO-2701 -s"New title" --no-input

# WORKS:
echo "" | jira issue comment add PO-2701 --body-adf '...' --no-input
```

Every write command needs `echo "" |` prefix to avoid hanging. This is a regression —
agents must always pipe stdin for writes.

### 4. Comment Management — FIXED

Full CRUD is now available:

| Operation | Command | Tested |
|-----------|---------|--------|
| Add | `jira issue comment add KEY "text"` | ✓ |
| Add (ADF) | `jira issue comment add KEY --body-adf '{...}'` | ✓ |
| Edit | `jira issue comment edit KEY COMMENT-ID --body-adf '{...}'` | ✓ |
| Delete | `jira issue comment delete KEY COMMENT-ID` | ✓ |
| List | `jira issue comment list KEY --plain` | ✓ |
| List (JSON) | `jira issue comment list KEY --raw` | ✓ |

### 5. Attachments — FIXED

New `attach` subcommand:

```bash
echo "" | jira issue attach add PO-2701 /tmp/test_attach.txt
# ✓ Attachment "test_attach.txt" uploaded to PO-2701 (id: 145058, size: 13 bytes)

echo "" | jira issue attach delete 145058
# ✓ Attachment 145058 deleted
```

Note: `attach delete` takes only ATTACHMENT-ID (no issue key).

### 6. Custom Fields — IMPROVED

New `--custom-json` flag accepts raw JSON values:

```bash
echo "" | jira issue edit PO-2701 --custom-json 'customfield_10072={"type":"doc",...}' --no-input
```

Tested: the JSON is sent correctly, but the test field (`customfield_10072`) returned
"cannot be set — not on the appropriate screen". This is a Jira screen configuration
issue, not a CLI bug. The `--custom-json` mechanism itself works.

The old `--custom` flag remains for simple string values.

### 7. Transition Discovery — FIXED

Invalid transition now shows available states:

```bash
echo "" | jira issue move PO-2701 "Nonexistent State"
# ✗ Error: invalid transition state "Nonexistent State"
# Available states for issue PO-2701: 'Team Backlog', 'Pause',
#   'Release Branch Before QA', 'Resume Development', 'Resume Review',
#   'Resume QA', 'Done'
```

### 8. JQL ORDER BY — NOT FIXED

```bash
jira issue list --plain -q"assignee = currentUser() AND status = 'In Progress' ORDER BY updated DESC"
# Error: Expecting ',' but got 'ORDER'. (line 1, character 92)
```

The CLI still appends its own `ORDER BY created DESC`. Workaround: use `--order-by`
flag instead of including ORDER BY in JQL:

```bash
jira issue list --plain -q"assignee = currentUser() AND status = 'In Progress'" --order-by updated
```

**However**, `--order-by` with `-q` returned no results in testing (may be a bug with
how the flag interacts with raw JQL).

### 9. Output Formats — FIXED

### 10. Field Selection — IMPROVED

## Remaining Issues for Agent Use

### Critical

1. **Stdin hang on writes**: Every write command hangs unless stdin is piped.
   Agents must prefix all write calls with `echo "" |`. This is fragile and
   easy to forget.

2. **Plain text `-b` flag is broken**: Since the CLI now uses API v3,
   `-b "plain text"` fails with ADF validation error. Only `--body-adf` works.
   The flag exists but is a trap.

### Medium

3. **JQL ORDER BY still mangled**: Must use `--order-by` flag instead.
4. **Debug leaks credentials**: Avoid `--debug` in agent contexts.
5. **`view` has no field selection**: Still fetches everything, no `--fields` flag.

### Low

6. **No bulk operations**: Still one issue per CLI call.
7. **ANSI in `--plain` footer**: Minor ANSI codes leak in `view --plain` output.

## Agent Usage Pattern (v1.8.0)

```bash
# READS — same as before
jira issue view PO-2701 --plain
jira issue view PO-2701 --raw
jira issue list --plain -q"..." --paginate 20
jira issue list --csv --columns KEY,STATUS,SUMMARY

# WRITES — always pipe stdin, always use --body-adf
echo "" | jira issue edit KEY --body-adf '{"type":"doc",...}' --no-input
echo "" | jira issue edit KEY -s"New summary" --no-input
echo "" | jira issue comment add KEY --body-adf '{"type":"doc",...}' --no-input
echo "" | jira issue comment edit KEY COMMENT-ID --body-adf '...'
echo "" | jira issue comment delete KEY COMMENT-ID
echo "" | jira issue attach add KEY /path/to/file
echo "" | jira issue attach delete ATTACHMENT-ID
echo "" | jira issue move KEY "In Progress"
```
