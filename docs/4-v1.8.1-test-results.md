## Detailed Test Results

All tests used dummy ticket `PO-2840` (created, tested, then archived).

### 1. ADF Description Writes ‚Äî PASSED

```bash
jira issue edit PO-2840 --body-adf '{"type":"doc","version":1,"content":[...]}' --no-input < /dev/null
# ‚úì Issue updated
```

Verified via `--raw`: ADF structure preserved with headings, bold, italic marks.

### 2. API Version ‚Äî PASSED

Debug output confirms v3 for both reads and writes:
```
GET /rest/api/3/issue/PO-2840 HTTP/1.1
PUT /rest/api/3/issue/PO-2840 HTTP/1.1
```

### 3. Interactive Prompts / Stdin Hang ‚Äî NOT FIXED

Despite the changelog claiming a fix, write commands still hang without stdin redirection:

```bash
# HANGS (no stdin pipe):
jira issue edit PO-2840 -s"New title" --no-input
# ‚Üí Sent to background after 8s, no output

# WORKS (with < /dev/null):
jira issue edit PO-2840 -s"New title" --no-input < /dev/null
# ‚úì Issue updated
```

**Caveat with `< /dev/null`**: When only editing the summary (no `-b` or `--body-adf`),
`< /dev/null` causes the CLI to read empty stdin as a body, sending an empty ADF
document that Jira rejects:

```bash
jira issue edit PO-2840 -s"New title" --no-input < /dev/null
# Error: description: Operation value must be an Atlassian Document

echo "" | jira issue edit PO-2840 -s"New title" --no-input
# ‚úì Issue updated ‚Äî works because echo provides a newline, not EOF
```

**Agent rule**: Use `echo "" |` for summary-only edits, `< /dev/null` when `-b`
or `--body-adf` is also provided.

### 4. Plain Text `-b` Flag ‚Äî FIXED

The `-b` flag now auto-wraps plain text in an ADF paragraph:

```bash
jira issue edit PO-2840 -b "Plain text body" --no-input < /dev/null
# ‚úì Issue updated
```

Verified via `--raw`: description is a valid ADF `doc` with a single paragraph node.

**Note**: `-b` on `create` is broken ‚Äî the auto-wrap sends an ADF object but the create
endpoint rejects it with "Operation value must be a string". Omit `-b` on create
or use the REST API for descriptions at creation time.

## Remaining Issues

### Critical

1. **Stdin hang on writes**: Write commands still hang without stdin pipe. Agents
   must use `echo "" |` or `< /dev/null` for every write command.

2. **`< /dev/null` breaks summary-only edits**: When no body flag is given,
   `< /dev/null` causes the CLI to send an empty ADF body. Use `echo "" |` instead.

3. **`create -b` is broken**: The auto-wrap applies to `edit` but not `create`.
   Creating an issue with `-b "text"` fails with "Operation value must be a string".

### Low

4. **No `--body-adf` on `create`**: Must create without a body and then `edit`
   to add the description, or use REST API directly.

5. **No bulk operations**: Still one issue per CLI call.

6. **No webhook/event support**: Cannot manage webhooks or poll for changes.

## Agent Usage Pattern (v1.8.1)

```bash
# READS ‚Äî no stdin workaround needed
jira issue view KEY --plain
jira issue view KEY --raw
jira issue view KEY --fields summary,status,description --raw
jira issue list --plain -q"..." --paginate 20
jira issue list --csv --columns KEY,STATUS,SUMMARY -q"..."
jira issue comment list KEY --raw

# WRITES ‚Äî always pipe stdin
echo "" | jira issue edit KEY --body-adf '{"type":"doc",...}' --no-input
echo "" | jira issue edit KEY -b "plain text description" --no-input
echo "" | jira issue edit KEY -s"New summary" --no-input
jira issue comment add KEY "plain text" --no-input < /dev/null
jira issue comment add KEY --body-adf '{"type":"doc",...}' --no-input < /dev/null
jira issue comment edit KEY COMMENT-ID --body-adf '...' < /dev/null
jira issue comment delete KEY COMMENT-ID < /dev/null
jira issue attach add KEY /path/to/file < /dev/null
jira issue attach delete ATTACHMENT-ID < /dev/null
jira issue move KEY "In Progress" < /dev/null

# CREATE ‚Äî without body (add description via edit afterward)
jira issue create -p PROJ -t Task -s "Summary" --no-input < /dev/null --raw
```
